Top List of SAS Complaints

Posted on 11 Jul 2014

SAS can do a lot of things, but there are top things that I find very annoying

  1. Non-separability of windows

    I have multiple monitors that I work with. I would like to be able to separate my SAS windows to appear on different monitors. For example, code on one monitor and output on the other

  2. Non-integration of output

    This one really gets to me. I have no idea why the SAS community has tolerated this deficiency for so long.

  3. Dangers of many-to-many merges
  4. You can't have your data set open when running a data step
  5. You can't code when a procedure is running

Introduction to SAS

Posted on 16 Feb 2014

My first introduction to a SAS data set was via an example of directly reading a data set.

DATA Leukemia;
  INPUT IdNum Weeks Status Group Sex $ Age ;
  DATALINES;
      1  6  0  1  F  10
      2  6  1  1  M  12
;
RUN;

The $ after Sex defines the variable Sex to be a character variable.

Where are my files?

This can be an early point of frustration to the new SAS user. Someone has sent you a SAS file FinalData.sas7bdat, and you have saved it to a directory on your computer. You start SAS and try to work out how to open your file.

libname MyDir "C:\some\path\to\my\data" ;

In the SAS log you should get a message to indicate success:

NOTE: Libref MyDir successfully assigned as follows:
      Engine:        V9
      Physical Name: C:\some\path\to\my\data

DATA MissedVisaPayment;
    SET MissedVisaPayment;
    RETAIN num_missed;
    BY id;
    IF FIRST.id THEN num_missed = 0;
    IF missed_payment = 0 THEN
       num_missed = num_missed;
    ELSE IF missed_payment = 1 THEN
       num_missed = num_missed + 1;
RUN;
